Testing Proxy with PAC files

How to use this?
Run the below commands as it is in the Agent machine, change PAC-filenames in URL, Credentials are dependent
1. To Configure
catchpoint proxy --type=PAC --host=https://playwright-karthik.github.io/qatesting/p1.pac --user=proxyuser --password=cpuser123
2. To Remove
catchpoint proxy --type=DIRECT
catchpoint restart
===============
PROXY SETTINGS:
"Proxy Authentication" support from Agent node
configuration:
(Only Basic Auth is verified, No resource for Ntlm/ Digest Proxy) (Puppeteer dont support NTLM)

"PAC proxy" support via Config Key as below,
-Puppeteer test with proxy runs by default.

-To run a playwright test, user need to add a config key and set it to TRUE
PlaywrightTestMonitor_AllowHeadlessNewMode=true
vi etc/catchpoint.d/TxEng.conf

//otherwise you see below error
[51042] - A tunnel connection through the proxy could not be established.

Setting BasicProxy:
1) Non-Authenticated Proxy: [241]
[forcing all requests to pass through 241]

catchpoint proxy --type=STATIC --host=192.168.128.241 --port=3128

2) Authenticated Proxy: [243]
[forcing all requests to pass through 243: Needs Uname and Pword]

catchpoint proxy --type=STATIC --host=192.168.128.243 --port=50000 --user=proxyuser --password=cpuser123

3) Setting PAC-Authenticated:
[forcing all requests to pass through 243]
catchpoint proxy --type=PAC --host=https://playwright-karthik.github.io/qatesting/p6.pac --user=proxyuser --password=cpuser123
3) Setting PAC-Non Authenticated:
[forcing all requests to pass through 241]
catchpoint proxy --type=PAC --host=https://playwright-karthik.github.io/qatesting/p7.pac
