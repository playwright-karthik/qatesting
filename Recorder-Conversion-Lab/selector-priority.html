How To Use Basic Interactions Selector Priority Edge Cases

Expected Fallback Order

If advanced selectors exist, Agent should use only advanced selectors in priority order. Otherwise the normal order is:

role + name -> label -> test id -> text -> css id -> css stable ancestor -> css class -> xpath -> tag fallback

  1. Interact with every row from top to bottom.
  2. Run the generated JSON and script through the analyser.
  3. Open the Selector Priority tab and confirm the actual script order matches expected order.

Role + Name

Expected first selector: role + accessible name.

Label

Expected first selector: label.

Test ID

Expected selector: test id when role/name and label are absent.

Text

Expected selector: visible text.

CSS ID

Expected selector: css id.

Stable Ancestor

Expected selector: stable ancestor CSS path.

CSS Class

Expected selector: unique class.

XPath

Expected selector: XPath fallback.

Tag Fallback

Expected selector: tag fallback when it is the only available selector.

Advanced Selector Candidate

Use this row when the recorder emits advanced selectors. If advanced selectors are present in JSON, normal selector fallbacks should not appear in script.
